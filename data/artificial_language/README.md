# Data
## Artificial Languages
Artificial languages are generated in a controlled setup to isolate the influence of semantic structure, anchor tokens, and crosslingual transfer behavior. Instead of relying on naturally occurring corpora, the vocabulary and grammar are synthetically constructed while preserving selected semantic relations from external knowledge resources.

The artificial languages are designed to:
- minimize uncontrolled corpus-specific biases,
- enable reproducible experiments,
- explicitly manipulate semantic overlap and anchor ratios,
- study multilingual transfer under controlled conditions.
### Knowledge Base
Semantic structure is derived from external lexical and knowledge resources.

Possible resources include:

- **WordNet**
  - provides hierarchical lexical-semantic relations,
  - useful for:
    - hypernymy (`dog → animal`),
    - hyponymy,
    - synonym sets (synsets),
    - semantic distance.

The extracted semantic graph is used as the backbone of the artificial language. Surface forms are replaced while preserving underlying semantic topology.

Example:

| Natural Concept | Artificial Token |
|---|---|
| dog | tavor |
| animal | elkan |
| bark | ruvan |

Semantic relations remain unchanged.

This allows experiments to separate:
- semantic structure,
- lexical form,
- multilingual overlap,
- anchor preservation.

### Artificial WordNet Sampler

This project builds a small artificial-language WordNet from the original
NLTK WordNet.

The main script is:

```bash
build_artificial_wordnet.py
```

It samples a dense subgraph from WordNet, assigns every sampled synset a unique
artificial word, preserves the semantic relations between sampled synsets, and
exports the result as JSON.

#### Method
WordNet is treated as a graph:

```text
synset = node
semantic relation = edge
```

For example:

```text
dog.n.01 --hypernym--> canine.n.02
```

The sampler builds the artificial WordNet in three layers:

```text
base   -> basic concepts
layer1 -> concepts directly related to base concepts
layer2 -> concepts related to base + layer1
```

Instead of random sampling, the script prefers nodes that are more connected to
the current graph. A candidate synset receives a higher score if it is connected
to already selected nodes, and a smaller score if it is connected to other
candidates. This makes the resulting graph denser and reduces isolated nodes.

#### Artificial Words

Each sampled WordNet synset is mapped to one artificial word.

Artificial words are generated by:

1. Sampling characters from a Unicode range.
2. Sampling word length from a normal distribution.
3. Storing generated words in a set to guarantee uniqueness.

The default Unicode range is:

```text
U+A000..U+A4C6
```

This is the Yi Syllables block. It is used because the characters are visible,
numerous, and visually distinct from ordinary Latin text.

#### Usage

Install the dependency:

```bash
python3 -m pip install nltk
```

Run the sampler:

```bash
python3 build_artificial_wordnet_unicode.py \
  --base-size 100 \
  --layer1-size 5000 \
  --layer2-size 1000 \
  --unicode-start 0xA000 \
  --unicode-end 0xA4C6 \
  --word-length-mean 4.5 \
  --word-length-std 1.25 \
  --word-length-min 2 \
  --word-length-max 9 \
  --output artificial_wordnet_unicode.json
```

#### Output Format

The output JSON contains:

```json
{
  "metadata": {},
  "summary": {},
  "nodes": [],
  "edges": [],
  "mappings": {}
}
```

##### `nodes`

Each node is an artificial-language synset:

```json
{
  "id": "ꀀꂊꃁ",
  "original_id": "dog.n.01",
  "layer": "layer1",
  "pos": "n",
  "original_lemmas": ["dog", "domestic_dog"],
  "definition": "...",
  "examples": []
}
```

##### `edges`

Each edge preserves a WordNet semantic relation:

```json
{
  "source": "ꀀꂊꃁ",
  "relation": "hypernym",
  "target": "ꁧꄂ",
  "source_original": "dog.n.01",
  "target_original": "canine.n.02"
}
```

This means:

```text
artificial_word_for(dog.n.01) --hypernym--> artificial_word_for(canine.n.02)
```

##### `mappings`

The mappings connect original WordNet synsets and artificial words:

```json
{
  "original_to_artificial": {
    "dog.n.01": "ꀀꂊꃁ"
  },
  "artificial_to_original": {
    "ꀀꂊꃁ": "dog.n.01"
  }
}
```

#### Summary Metrics

The `summary` field reports graph quality:

```json
{
  "node_count": 3401,
  "edge_count": 9669,
  "average_degree": 5.686,
  "directed_density": 0.00083617,
  "isolated_node_count": 0
}
```

Important fields:

- `node_count`: number of artificial synsets.
- `edge_count`: number of preserved semantic relations.
- `average_degree`: average number of connections per node.
- `isolated_node_count`: number of nodes with no edges.
- `layer_counts`: number of nodes sampled into each layer.
- `relation_counts`: number of edges by relation type.

For WordNet-style graphs, `average_degree` and `isolated_node_count` are more
useful than raw graph density, because WordNet is naturally sparse.

#### Notes

The script does not simply translate English words. It maps WordNet concepts
to artificial words:

```text
dog.n.01 -> artificial word
bank.n.01 -> artificial word
bank.n.09 -> different artificial word
```

This avoids ambiguity from English words with multiple senses.
